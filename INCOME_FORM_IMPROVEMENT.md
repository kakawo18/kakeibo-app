# 収入フォームの改善

## 変更内容

### 支払方法フィールドの条件付き表示

**変更前**:
- 収入・支出どちらの場合も支払方法フィールドが表示される
- 収入入力時に不要なフィールドが表示されていた

**変更後**:
- 支出の場合のみ支払方法フィールドを表示
- 収入の場合は支払方法フィールドを非表示

## 実装の詳細

### 条件付きレンダリング

```typescript
{/* 支払方法は支出の場合のみ表示 */}
{form.values.type === 'expense' && (
  isMobile ? (
    <NativeSelect
      label="支払方法"
      // ... 省略
    />
  ) : (
    <Select
      label="支払方法"
      // ... 省略
    />
  )
)}
```

### 自動クリア処理

収入⇔支出を切り替えた際に、支払方法を自動的にクリア：

```typescript
<SegmentedControl
  onChange={(value) => {
    form.setFieldValue('type', value as 'income' | 'expense');
    form.setFieldValue('category', '');
    form.setFieldValue('subcategory', '');
    // 収入に切り替えた場合は支払方法をクリア
    if (value === 'income') {
      form.setFieldValue('paymentMethod', '');
    }
  }}
/>
```

## メリット

### 1. UIのシンプル化
- 収入入力時に不要なフィールドが表示されない
- フォームがすっきりして見やすい
- 入力項目が減り、入力が速くなる

### 2. ユーザー体験の向上
- 混乱を防ぐ（収入に支払方法は不要）
- 入力ミスの防止
- 直感的な操作

### 3. データの整合性
- 収入データに不要な支払方法が保存されない
- データベースの整合性が向上

## フォームの表示項目

### 支出入力時
1. 種別（支出/収入）
2. 金額
3. カテゴリ
4. サブカテゴリ（任意）
5. **支払方法（任意）** ← 表示
6. 日付
7. メモ（任意）

### 収入入力時
1. 種別（支出/収入）
2. 金額
3. カテゴリ
4. サブカテゴリ（任意）
5. ~~支払方法~~ ← 非表示
6. 日付
7. メモ（任意）

## 関連する変更

この変更は以下の改善と組み合わせて実装されています：

1. **NativeSelectの採用**（モバイル）
   - 支払方法選択がスムーズ
   - 長いリストでも安定

2. **レスポンシブ対応**
   - モバイル: NativeSelect
   - デスクトップ: Select（検索機能付き）

3. **フォーム最適化**
   - タップ領域48px
   - フォントサイズ16px
   - ズーム防止

## まとめ

収入入力時に支払方法フィールドを非表示にすることで、フォームがシンプルになり、ユーザー体験が向上しました。支出⇔収入の切り替え時には自動的に支払方法がクリアされるため、データの整合性も保たれます。
