# 家計簿アプリ - ユーザーガイド

## サーバー起動方法

### 基本起動
```bash
npm run dev
```

### WSL環境での外部アクセス対応起動
```bash
npm run dev -- --hostname 0.0.0.0
```

**アクセスURL**:
- WSL内: http://localhost:3000
- Windows環境: http://172.23.222.42:3000 (IPアドレスは環境により変動)

### その他のコマンド
```bash
# プロダクションビルド
npm run build

# プロダクションサーバー起動
npm run start

# 型チェック
npm run type-check

# リント
npm run lint
```

## アプリの使い方

### 1. 初回セットアップ

1. **アカウント作成**: メールアドレスとパスワードでアカウント作成
2. **ログイン**: 作成したアカウントでログイン

### 2. 基本的な取引入力

#### 収入の入力
1. **「取引を追加」ボタン**をクリック
2. **収入/支出**で「収入」を選択
3. **金額**を入力（円単位）
4. **カテゴリ**を選択：
   - 給与 → 給与・ボーナス
   - その他
5. **日付**をカレンダーから選択（デフォルト：今日）
6. **説明**（オプション）
7. **「追加」ボタン**で保存

#### 支出の入力（現金支払い）
1. **「取引を追加」ボタン**をクリック
2. **収入/支出**で「支出」を選択
3. **金額**を入力
4. **カテゴリ**を選択：
   - 食費 → 食費・飲み会費
   - 光熱費 → 電気代・ガス代
   - 固定費 → 家賃・投資
   - 交通費、趣味代、旅行代、医療費
   - カード引き落とし → 各カード会社
   - その他
5. **支払方法**で「現金」を選択
6. **日付**と**説明**を入力
7. **「追加」ボタン**で保存

#### クレジットカード支払い
1. 支出入力で**支払方法**を以下から選択：
   - 三井住友カード
   - 三菱UFJカード
   - amazonカード
   - EPOSカード
   - 楽天カード
2. その他の項目は現金支払いと同様

**重要**: カード支払いは**翌月の残高**に影響します（実際の引き落とし時期を反映）

### 3. 月次ナビゲーション

#### 月の切り替え
- **左上の「2025年06月」**をクリックして月を選択
- **前月/次月ボタン**で移動
- **過去2年、未来1年**まで選択可能

#### 月別データ表示
- **収支内訳**: 選択月のデータのみ
- **取引履歴**: 選択月の取引のみ
- **残高推移**: 全期間の累積データ

### 4. データ管理

#### 取引の編集・削除
1. **取引履歴**で対象の取引をクリック
2. **編集**: 内容を修正して「更新」
3. **削除**: 「削除」ボタンで削除

#### CSVエクスポート・インポート
1. **右上のメニュー**（三点アイコン）をクリック
2. **エクスポート**: 全取引データをCSVでダウンロード
3. **インポート**: CSVファイルからデータを復元

### 5. 家計簿の見方

#### サマリーカード
- **今月の収入**: 選択月の収入合計
- **今月の支出**: 選択月の支出合計
- **収支差額**: 収入 - 支出
- **現在の残高**: 累積残高

#### グラフ表示
- **残高推移**: 月ごとの残高変化（折れ線グラフ）
- **収入内訳**: 収入カテゴリ別割合（円グラフ）
- **支出内訳**: 支出カテゴリ別割合（円グラフ）

#### 先月比較
各カテゴリで前月との比較を％で表示

### 6. クレジットカード管理の理解

#### カード支払いの流れ
1. **6月にカードで1000円支払い**
   - 支出: 1000円計上
   - 6月収支: -1000円
   - 6月残高: 変化なし

2. **7月に自動引き落とし**
   - 支出: 計上しない（既に6月で計上済み）
   - 7月収支: 影響なし
   - 7月残高: -1000円

#### 残高の正確性
- **実残高**: 銀行口座の実際の残高を反映
- **支出統計**: カード支払い時点での支出を記録
- **キャッシュフロー**: 実際のお金の動きを正確に追跡

### 7. トラブルシューティング

#### データが表示されない
- ネットワーク接続を確認
- ブラウザを再読み込み
- 別の月に切り替えて戻る

#### 計算が合わない
- カード支払いは翌月残高に影響することを確認
- 取引のカテゴリと支払方法を確認

#### CSVインポートエラー
- CSV形式が正しいかチェック
- 日付形式が正しいかチェック
- ファイルサイズが適切かチェック

## 設定（カスタマイズ）

ヘッダーの歯車アイコン（モバイルはメニュー内の「設定」）から設定ページを開けます。
設定はアカウントごとにクラウド保存され、開いているダッシュボードへ即時反映されます。

### カテゴリ管理
- 支出/収入カテゴリの追加・編集・削除・並べ替え
- サブカテゴリの管理と、カテゴリごとの表示色（パレットから選択）
- **役割**の付与: 貯蓄率・投資額などの集計はカテゴリ名ではなく役割で判定されます
  - 給与収入: 貯蓄率計算の年間収入に含める
  - 投資: 支出から除外し、年間投資額として集計
  - 立替金 / 立替回収: 収支から除外
  - カード引き落とし: 支出集計に含めない（カード支払い時に計上済みのため）
  - 支出ペース除外: 支出ペースチャートの対象から外す（家賃など）
- カテゴリを削除・リネームしても既存の取引は元のカテゴリ名のまま残ります

### 支払い方法・カード管理
- 支払方法の追加・削除
- カードごとのポイント還元率（獲得ポイントの計算に使用）
- 「現金扱い」スイッチ: オンにすると支払い時に残高から即時引き落とされる扱いになります

### 月間予算
- 支出ペースチャートの基準となる1ヶ月の予算を設定できます

### 定期取引
- 設定ページから定期取引の管理画面を開けます

## 推奨使用方法

1. **日々の記録**: 支出発生時にすぐ入力
2. **月末チェック**: 月末に残高と実際の口座残高を照合
3. **定期バックアップ**: 月1回CSVエクスポートでバックアップ
4. **分析活用**: 月次比較で支出傾向を把握

## 注意事項

- **データ同期**: Firestoreによる自動同期（複数デバイス対応）
- **セキュリティ**: 個人使用想定、基本的なセキュリティのみ
- **データ保存**: 無制限保存（可能な限り長期保存）
- **通貨単位**: 円のみ（小数点切り捨て）
