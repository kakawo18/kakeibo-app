# 家計簿アプリ - プロジェクト構造説明

このドキュメントは、家計簿アプリの各ファイルとディレクトリの役割を説明します。

## 📁 プロジェクト構造

```
kakeibo-app/
├── src/                      # ソースコード
│   ├── app/                  # Next.js App Router
│   ├── components/           # Reactコンポーネント
│   ├── contexts/             # React Context（状態管理）
│   ├── hooks/                # カスタムフック
│   ├── lib/                  # 外部ライブラリ設定
│   ├── types/                # TypeScript型定義
│   └── utils/                # ユーティリティ関数
├── public/                   # 静的ファイル
└── 設定ファイル群
```

---

## 🎯 src/app/ - Next.js App Router

Next.js 14のApp Routerを使用したアプリケーションのエントリーポイント。

### `layout.tsx`
- **役割**: アプリ全体のレイアウト定義
- **内容**:
  - Mantineのテーマ設定（ダークモード対応）
  - 認証プロバイダーのラップ
  - PWAのメタデータ設定
  - グローバルなスタイル読み込み

### `page.tsx`
- **役割**: トップページ（ダッシュボード）
- **内容**:
  - 認証チェック
  - ログイン画面またはダッシュボードの表示切り替え

### `globals.css`
- **役割**: グローバルスタイル定義
- **内容**:
  - Tailwind CSS設定
  - ダークモードのカラー定義
  - PWA用の入力フィールド最適化
  - アニメーション定義
  - タッチ操作の最適化

### `favicon.ico`
- **役割**: ファビコン（ブラウザタブのアイコン）

---

## 🧩 src/components/ - Reactコンポーネント

### 📊 charts/ - グラフコンポーネント

#### `PieChart.tsx`
- **役割**: 円グラフの表示
- **用途**: カテゴリ別支出の割合を視覚化
- **機能**:
  - Mantine Chartsを使用
  - レスポンシブ対応
  - カテゴリ別の色分け

#### `LineChart.tsx`
- **役割**: 折れ線グラフの表示
- **用途**: 月別の収支推移を視覚化
- **機能**:
  - 収入・支出・収支の3つの線
  - 月ごとの比較
  - ツールチップ表示

### 📝 forms/ - フォームコンポーネント

#### `TransactionForm.tsx`
- **役割**: 取引の追加・編集フォーム
- **機能**:
  - 収入/支出の切り替え
  - カテゴリ・サブカテゴリ選択
  - 日付・金額・説明の入力
  - モバイル最適化（16pxフォント、48pxタップ領域）
  - バリデーション

### 🔄 recurring/ - 定期取引コンポーネント

#### `RecurringTransactionManager.tsx`
- **役割**: 定期取引の一覧管理
- **機能**:
  - 定期取引の一覧表示
  - 編集・削除
  - 有効/無効の切り替え
  - 次回実行日の表示

#### `RecurringTransactionForm.tsx`
- **役割**: 定期取引の追加・編集フォーム
- **機能**:
  - 実行頻度の設定（毎月、毎週、毎年）
  - 実行日の指定
  - 取引内容の設定
  - モバイル最適化

#### `RecurringTransactionConfirm.tsx`
- **役割**: 定期取引の実行確認モーダル
- **機能**:
  - 実行予定の定期取引を表示
  - 一括実行または個別スキップ
  - 実行結果の通知

#### `RecurringTransactionNotice.tsx`
- **役割**: 定期取引の通知バッジ
- **機能**:
  - 実行待ちの定期取引数を表示
  - クリックで確認モーダルを開く
  - アニメーション付き

### 🖥️ ui/ - UIコンポーネント

#### `DashboardContent.tsx`
- **役割**: ダッシュボードのメインコンテンツ
- **機能**:
  - 月別収支サマリー表示
  - 年月選択（スワイプ対応）
  - グラフ表示
  - 取引一覧
  - 各種モーダルの管理
  - レスポンシブレイアウト

#### `TransactionList.tsx`
- **役割**: 取引履歴の一覧表示
- **機能**:
  - 日付ごとにグループ化
  - 編集・削除機能
  - カテゴリアイコン表示
  - 収入/支出の色分け
  - モバイル最適化

#### `MobileCalendar.tsx`
- **役割**: モバイル用カレンダー
- **機能**:
  - 日付選択
  - 年月移動（スワイプ対応）
  - 日別収支の表示
  - クイック選択（今日、昨日、一昨日）
  - 横画面対応
  - 日別取引詳細表示

#### `LoginForm.tsx`
- **役割**: ログインフォーム
- **機能**:
  - メール/パスワード認証
  - Googleログイン
  - 新規登録
  - PWA対応の入力フィールド

#### `CSVImportExport.tsx`
- **役割**: CSV入出力機能
- **機能**:
  - 取引データのCSVエクスポート
  - CSVファイルのインポート
  - データ検証
  - エラーハンドリング

#### `CardRewardsDisplay.tsx`
- **役割**: クレジットカードポイント表示
- **機能**:
  - カテゴリ別のポイント還元額計算
  - カード別の集計
  - 月別比較

#### `VersionDisplay.tsx`
- **役割**: アプリバージョン表示
- **機能**:
  - package.jsonからバージョン取得
  - ダッシュボード下部に表示

#### `Layout.tsx`
- **役割**: 共通レイアウトコンポーネント
- **機能**:
  - ヘッダー
  - ナビゲーション
  - フッター

### `PWAInstaller.tsx`
- **役割**: PWAインストールプロンプト
- **機能**:
  - インストール可能時にバナー表示
  - インストールボタン
  - 自動非表示

## 🔌 src/contexts/ - React Context

### `AuthContext.tsx`
- **役割**: 認証状態の管理
- **機能**:
  - ユーザー認証状態の保持

## 🪝 src/hooks/ - カスタムフック

### `useTransactions.ts`
- **役割**: 取引データの管理
- **機能**:
  - Firestoreからの取引データ取得
  - リアルタイムリスナー
  - 取引の追加・更新・削除
  - 月別フィルタリング
  - キャッシュ管理

### `useRecurringTransactions.ts`
- **役割**: 定期取引の管理
- **機能**:
  - 定期取引データの取得
  - リアルタイムリスナー
  - 定期取引の追加・更新・削除
  - 実行判定ロジック
  - 自動実行処理

### `firebase.ts`
- **役割**: Firebase初期化と設定

## 📘 src/types/ - TypeScript型定義

  - `RecurringTransaction`: 定期取引データの型
  - `Category`: カテゴリの型
  - `MonthlyData`: 月別集計データの型

### `calculations.ts`
- **役割**: 集計・計算ロジック
- **機能**:
  - 月別収支の計算
  - カテゴリ別集計
  - グラフデータの生成
  - 前月比較

### `dateUtils.ts`

  - タイムゾーン対応
  - 日付比較

### `csvUtils.ts`
- **役割**: CSV入出力ユーティリティ
- **機能**:
  - 取引データのCSV変換
  - CSVパース
  - データ検証
  - エンコーディング処理

### `cardRewards.ts`
- **役割**: クレジットカードポイント計算
- **機能**:

  - カード別集計

## ⚙️ 設定ファイル

### `package.json`
- **役割**: プロジェクト設定と依存関係
- **内容**:
  - 依存パッケージ
  - スクリプト定義
  - バージョン情報

### `next.config.ts`
- **役割**: Next.js設定
- **内容**:
  - ビルド設定
  - PWA設定
  - 環境変数

### `tsconfig.json`
- **役割**: TypeScript設定
- **内容**:
  - コンパイラオプション
  - パスエイリアス
  - 型チェック設定

### `.env.local`
- **役割**: 環境変数（ローカル）
- **内容**:
  - Firebase設定
  - APIキー
  - **注意**: Gitにコミットしない

### `.gitignore`
- **役割**: Git管理対象外ファイルの指定
- **内容**:
  - node_modules
  - .env.local
  - ビルド成果物

## 🎨 技術スタック

### フロントエンド
- **Next.js 14**: React フレームワーク（App Router）
- **TypeScript**: 型安全な開発
- **Mantine UI**: UIコンポーネントライブラリ
- **Framer Motion**: アニメーション
- **Tailwind CSS**: ユーティリティファーストCSS

### バックエンド
- **Firebase Authentication**: ユーザー認証
- **Cloud Firestore**: NoSQLデータベース
- **Firebase Hosting**: ホスティング（Vercel経由）

### PWA
- **next-pwa**: PWA対応
- **Service Worker**: オフライン対応
- **Web App Manifest**: インストール対応

## 🔄 データフロー

```
1. ユーザー操作
   ↓
2. コンポーネント（UI）
   ↓
3. カスタムフック（useTransactions, useRecurringTransactions）
   ↓
4. Firebase（Firestore）
   ↓
5. リアルタイムリスナー
   ↓
6. 状態更新
   ↓
7. UI再レンダリング
```
